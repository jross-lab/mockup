A plain-English summary of every update made over the past two days.

26 March 2026
Improvement
Control panel reorganised into clear sections
The left-hand input panel is now split into five named sections — Brand Assets, Brand & Challenge, Engagement & Discovery, Challenge Progress, and Content — each with a short description explaining where that information appears in the mockup. Fields are no longer one long undifferentiated list.
New
Character counters on text fields
The longer free-text fields (Challenge Title, Goal, In-Feed CTA, Reward, Description) now show a live character count next to the label (e.g. "32/60"). The counter turns orange when you're close to the limit so you know before you hit it.
Improvement
Field hints added so it's clear where each input appears
Several fields now show a small grey note next to their label — for example Goal says "Shown in the challenge info rows" and Description says "Shown in the About this Challenge section". No more guessing which field maps to which part of the screen.
Improvement
Image upload labels now describe where each image is used
The three image slots are now labelled "Hero Image", "Challenge Badge", and "Brand Logo", with a small caption underneath each explaining where it appears (e.g. "Banner behind badge", "Sponsor card").
Bug fix
Activity filter chips now show the correct sport icons
The Run / Ride / Swim / Walk / Hike filter chips on the Gallery screen were showing a plain circle instead of the proper sport icon. They now display the correct Strava activity icons. The same fix was applied to the identical chip bar visible in the background of the Takeover screen, which was also showing placeholder "Label" text instead of real activity names and icons.
25 March 2026
Improvement
Auto-formatting for dates and participant numbers
Date fields now automatically tidy up whatever format you type — whether that's "2026-05-01", "1 May 2026", or "May 1 2026" — and display it consistently as "May 1, 2026". Participant numbers are also auto-formatted with commas (e.g. 12480 becomes 12,480) when you leave the field.
Improvement
Activity type selector added to the control panel
You can now toggle which activity types apply to a challenge (Run, Ride, Swim, Walk, etc.) directly from the control panel using clickable pill buttons. The selected types update the "Qualifying Activities" line on all the detail screens.
Improvement
Activity icons updated to official Strava Brand Asset Library SVGs
The sport icons used throughout the tool (Run, Ride, Swim, Walk, Hike) were replaced with the official SVG icons from Strava's brand asset library, ensuring the mockups match what appears in the real Strava app.
New
Custom in-feed CTA button text is now editable
The call-to-action button on the Custom In-Feed screen (e.g. "Join the challenge") can now be customised from the control panel. Previously it was hardcoded and couldn't be changed.
Bug fix
Custom in-feed layout corrected to match Figma spec
The badge on the Custom In-Feed screen is now correctly top-aligned with the challenge title rather than centred alongside it, and the bike icon SVG was replaced with the correct version. A grey separator was also added below the ad unit to match the Figma design.
24 March 2026
New
Old Design screen added
A legacy version of the challenge detail screen has been added to the screen picker gallery. It's marked with a dashed border so it's easy to distinguish from the current design templates. Useful for showing before/after comparisons to clients.
Bug fix
PNG export now captures scrolled content correctly
When downloading a screen where you'd scrolled down inside the phone preview, the exported PNG was capturing the top of the screen rather than what you could see. This is now fixed — exports capture exactly the scroll position you had at the time of download.
Bug fix
Challenge badge clipping fixed on detail screens
The circular challenge badge that overlaps the bottom of the hero image was being cut off by the image container on some screens. It now sits correctly on top of the hero image as intended.
Bug fix
Divider lines removed from info rows
Thin grey divider lines were incorrectly appearing between the challenge info rows (dates, goal, reward) on several screens. These have been removed to match the Figma designs.
